
# Integrating the Creative Agent (Content Engine) with Existing Content Systems

## Current State Analysis

### Existing Content Generation Systems

| System | Edge Function | Used For |
|--------|--------------|----------|
| **AIContentButton** | `generate-website-content` | Inline ✨ buttons in forms (website headlines, email subjects, SMS, etc.) |
| **Social Media Console** | `generate-social-variations` | Batch social posts with platform-specific variations |
| **Blog Management** | `generate-blog-content` | Single and batch blog post generation |
| **Campaigns** | `generate-campaign-content` / `generate-campaign-series` | Email/SMS marketing campaigns |
| **Content Engine** | `content-engine` | Multi-channel unified generation (NEW) |

### Brand Consistency (Single Source of Truth)
All systems pull from the **`company_ai_content_profiles`** table for:
- Brand voice & tone
- Keywords to include/avoid
- Target audience
- Unique selling points

### Identified Gaps

1. **Content Engine generates but doesn't save** - Results are displayed but can't be pushed to Social, Campaigns, Blog, etc.
2. **Dashboard/Calendar tabs are empty placeholders**
3. **No cross-channel content tracking**
4. **Brand Voice settings tab says "coming soon"**

---

## Integration Plan

### Phase 1: Add "Use This Content" Actions

**Goal**: Allow Content Engine output to be saved directly to the appropriate systems.

**Changes to `MultiChannelGenerator.tsx`**:

Add action buttons to each generated result:

| Channel | Action | Destination |
|---------|--------|-------------|
| Social | "Schedule Post" | `scheduled_social_posts` table |
| Campaign | "Create Campaign" | `marketing_campaigns` table |
| Blog | "Save as Draft" | `scheduled_blog_posts` table |
| Website | "Apply to Website" | `smart_websites` table |
| SMS | "Save as Template" | `sms_templates` table |

```text
┌────────────────────────────────────────────┐
│  Generated Content Result Card             │
├────────────────────────────────────────────┤
│  [Copy] [Edit] [Schedule/Save to...]       │
│                                            │
│  Social → Redirects to Schedule Post       │
│  Campaign → Opens Campaign Form            │
│  Blog → Creates draft post                 │
│  Website → Updates website section         │
│  SMS → Saves as new template               │
└────────────────────────────────────────────┘
```

### Phase 2: Enable Brand Voice Settings Tab

**Goal**: Embed the AI Content Profile editor directly in Content Engine.

**Changes to `ContentEngineConsole.tsx`**:
- Import and render `AIContentProfileManager` in the "Brand Voice" tab
- Remove placeholder text

### Phase 3: Build Content Dashboard

**Goal**: Show content generated via Content Engine with basic metrics.

**New component**: `ContentEngineDashboard.tsx`
- Track content generation in a new `content_engine_history` table
- Display:
  - Total content pieces generated by channel
  - Recent generation history
  - Content that was saved/scheduled vs. discarded

### Phase 4: Build Unified Content Calendar

**Goal**: Cross-channel view of all scheduled content.

**New component**: `ContentEngineCalendar.tsx`
- Aggregate from:
  - `scheduled_social_posts`
  - `scheduled_blog_posts`
  - `marketing_campaigns` (with `scheduled_send_date`)
- Display in calendar format with channel color-coding

## Technical Details

### Database Changes
Add table for content engine tracking:
```sql
CREATE TABLE content_engine_history (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES companies(id),
  channel text NOT NULL,
  topic text NOT NULL,
  content jsonb NOT NULL,
  saved_to text, -- 'social_posts', 'campaigns', 'blog', etc.
  saved_id uuid, -- ID of the created record
  created_at timestamptz DEFAULT now(),
  created_by uuid REFERENCES profiles(id)
);
```
